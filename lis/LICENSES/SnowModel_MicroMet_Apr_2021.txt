From proposal:
  
Data Management and Open Source Code Plan

“The model source codes, documentation, configuration and analysis tools developed as part of this project will be made available to the community through github (www.github.com), a webbased source code repository. The code necessary to embed SnowModel into NASA-LIS will be distributed as part of NASA-LIS and ongoing SnowModel development will be performed on the same code base that will be used in LIS under a compatible open source license. The LIS source code is available as NASA Open Source software at the LIS Release web page https://lis.gsfc.nasa.gov/releases and github https://github.com/NASA-LIS/LISF. Model parameter and meteorological forcing data sets necessary to configure SnowModel in LIS will be provided with existing LIS input data sets and data readers. This will include the model configuration files and parameter settings for each of the SnowEx domains and the continental scale SEUP domain. Model outputs will be made available in summary form (temporally and/or spatially aggregated), because of the large anticipated data volume, raw data will be archived on NASA’s storage systems so that access can be provided as needed.”
